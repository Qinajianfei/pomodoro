# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 项目概述

单文件桌面番茄钟网页应用，双击 `pomodoro.html` 即可在浏览器中打开使用。无构建工具、无依赖。

## 部署 / 预览

在默认浏览器中打开：
```
powershell -Command "Start-Process 'd:\claude-nao\pomodoro.html'"
```

## 技术架构

- 纯静态 HTML 文件，CSS 和 JS 均内联
- SVG 环形进度条（stroke-dasharray/dashoffset 动画）
- Web Audio API 生成提示音（无需音频文件）
- Notification API 桌面通知
- localStorage 持久化统计数据（按日期区分）和用户设置
- 三种计时模式：work（专注）、shortBreak（短休）、longBreak（长休）
- 自动模式切换：完成 4 轮专注后自动进入长休

## 行为准则

以下准则旨在减少 LLM 常见编码错误。**权衡：偏谨慎而非速度，琐碎任务自行判断。**

### 1. 编码前思考

**不假设、不隐藏困惑、呈现权衡。**

实现之前：
- 明确陈述假设。不确定就问。
- 如果存在多种解释，呈现它们——不要默默选择。
- 如果有更简单的方案，说出来。必要时提出异议。
- 如果有不清楚的地方，停下来。指出困惑所在，发问。

### 2. 简洁优先

**最少代码解决问题。不写推测性代码。**

- 不添加要求之外的功能
- 不为一次性代码创建抽象
- 不添加未要求的"灵活性"或"可配置性"
- 不为不可能发生的场景做错误处理
- 200 行能写成的不要写成 50 行就重写

自问："资深工程师会觉得这过于复杂吗？" 如果是，简化。

### 3. 精准修改

**只碰必须碰的。只清理自己造成的混乱。**

编辑已有代码时：
- 不要"改进"相邻代码、注释或格式
- 不要重构没坏的东西
- 匹配现有风格，即使你有不同偏好
- 如果注意到无关的死代码，提出来——不要删

你的改动造成孤儿代码时：
- 删除因你的改动而不再使用的导入/变量/函数
- 不要删除已有的死代码，除非被要求

检验标准：每一行修改都应该能直接追溯到用户的需求。

### 4. 目标驱动执行

**定义成功标准。循环验证直到达成。**

将任务转化为可验证的目标：
- "添加验证" → "为无效输入编写测试，然后让它们通过"
- "修复 bug" → "写一个复现 bug 的测试，然后让它通过"
- "重构 X" → "确保重构前后测试通过"

多步骤任务先陈述简要计划：
```
1. [步骤] → 验证: [检查]
2. [步骤] → 验证: [检查]
3. [步骤] → 验证: [检查]
```

强有力的成功标准让你能自主循环执行。弱标准（"让它能跑"）需要不断澄清。
